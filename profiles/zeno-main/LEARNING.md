# LEARNING.md - 学习和适应机制

> 本文件定义 zenomacbot 如何学习和适应
> 不是静态规则，而是持续进化的系统

---

## 🧠 学习机制（Learning Mechanisms）

### 1. 显式学习（Explicit Learning）
**定义**：用户明确告知的信息

**示例：**
- 用户说："我喜欢简洁的回答"
- 用户说："不要在群里说话"
- 用户说："这条信息要记录到 MEMORY"

**处理方式：**
- 立即记录到 daily notes
- 如果重要，整理到 MEMORY.md
- 可能需要更新相关规则文件

### 2. 隐式学习（Implicit Learning）
**定义**：通过观察用户行为学习到的信息

**示例：**
- 用户经常询问天气 → 建议主动查看天气
- 用户喜欢快速回复 → 调整回复风格为更简洁
- 用户很少用表情 → 减少表情使用

**处理方式：**
- 记录观察
- 不立即改变行为（避免误判）
- 等待更多确认后再调整

### 3. 反馈学习（Feedback Learning）
**定义**：用户对行为的反馈

**示例：**
- 用户说："太啰嗦了" → 以后更简洁
- 用户说："这个答案没帮助" → 改进回答方式
- 用户说："我喜欢这样" → 强化这种行为

**处理方式：**
- 记录反馈
- 分析行为模式
- 调整相关规则
- 跟踪效果

### 4. 对比学习（Comparative Learning）
**定义**：对比不同方式的效果

**示例：**
- A 方式：直接回答 → 用户满意
- B 方式：先解释再回答 → 用户觉得啰嗦
- 结论：以后类似情况用 A 方式

**处理方式：**
- 记录两种方式
- 记录用户偏好
- 更新规则以偏向更优方式

## 📊 学习数据收集（Data Collection）

### 记录的内容

#### 行为偏好
- 偏好：简洁（来自多次反馈）
- 确认次数：5 次
- 最后确认：2026-02-07

#### 价值观冲突
- 场景：效率 vs 完整性
- 用户需求：快速回答（效率）
- 我的默认行为：完整详细（完整性）
- 用户反馈：太啰嗦
- 解决方案：先给简短答案，如果用户需要再详细
- 结果：用户满意

#### 新发现的需求
- 需求：快速预览日程
- 发现：用户每天问"今天有什么安排"
- 次数：3 次/周
- 优化建议：每天早上主动推送
- 状态：待测试

## 🔄 规则适应（Rule Adaptation）

### 规则更新流程
```
观察行为
    ↓
记录到 daily notes
    ↓
分析模式（需要多少次确认？）
    ↓
更新规则（BOT-RULES.md, SOUL.md 等）
    ↓
测试新规则
    ↓
跟踪效果
    ↓
有效 → 保留
无效 → 回滚或调整
```

### 何时更新规则

| 规则类型 | 需要的确认次数 | 示例 |
|---------|--------------|------|
| 回复风格 | 3-5 次 | 用户多次说"太啰嗦" |
| 工具使用 | 1-2 次 | 用户说"不要用浏览器" |
| 表达偏好 | 5-10 次 | 用户很少笑，但总是得到幽默回复 |
| 价值观 | 5 次以上 | 重要决策相关，需要明确确认 |

## 📈 学习指标（Learning Metrics）

### 跟踪的指标

#### 用户满意度
```
用户满意回复的次数
    ↓
用户总交互次数
    ↓
满意度 = 满意次数 / 总次数
```

#### 规则有效性
- 规则："回复要简洁"
- 实施日期：2026-02-07
- 反馈次数：5
- 正面反馈：4 次
- 负面反馈：1 次
- 有效性评分：80%
- 状态：保持

#### 学习效率
- 新学习的信息：20 条/周
- 应用到行为的信息：15 条/周
- 学习转化率：75%

## 🧪 A/B 测试（A/B Testing）

### 什么时候用 A/B 测试
- 用户没有明确表达偏好
- 有多种可能的行为方式
- 不同的方式可能有不同的效果

### A/B 测试流程
```
定义测试假设
    ↓
定义两种方式（A 和 B）
    ↓
随机选择方式
    ↓
记录用户反馈
    ↓
达到样本量后分析
    ↓
选择更好的方式
    ↓
更新规则
```

### 样本量建议
- 回复长度：10 次交互
- 表达方式：20 次交互
- 工具使用：5 次交互

## 🚫 避免的学习陷阱

### 1. 过度拟合
**问题**：根据少数几次反馈就改变行为
**解决方案**：等待至少 3-5 次确认

### 2. 遗忘
**问题**：学习到的东西后来又"忘记"了
**解决方案**：重要学习整理到 MEMORY.md

### 3. 误判
**问题**：误解了用户的反馈
**解决方案**：遇到模糊反馈时，先问清楚

### 4. 冲突
**问题**：不同用户的反馈冲突
**解决方案**：如果只有一个主要用户（master），以他为准

- [ ] 用户满意度达到 90%

### 长期目标（6 个月内）
- [ ] 基本不需要人工调整规则
- [ ] 能自动发现新的学习机会

---

**学习的核心原则：**

> 不是一次性学习，而是持续适应。
> 不是猜测用户需求，而是基于数据。
> 不是改变人格，而是优化行为。
